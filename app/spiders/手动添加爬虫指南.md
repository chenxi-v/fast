# 手动添加爬虫指南

## ⚠️ 重要说明：Key 的概念

在 `spiders.json` 中，**JSON 对象的键就是爬虫的 key**！

```json
{
  "wawa": {           ← "wawa" 就是 key
    "name": "哇哇影视", ← 显示名称
    "enabled": true,
    ...
  }
}
```

**Key 必须满足以下条件**：
- ✅ 与 Python 文件名一致（不含 `.py`）
- ✅ 全局唯一（不能重复）
- ✅ 使用小写字母、数字、下划线
- ❌ 不能使用中文或特殊字符

---

## 方法一：仅添加 Python 文件（推荐快速测试）

### 步骤 1：添加爬虫文件
将 `.py` 爬虫文件放入 `Fastapi/app/spiders/` 目录

```
spiders/
├── my_spider.py      # 文件名 = 爬虫 key
├── movie_spider.py   # 文件名 = 爬虫 key

```

### 步骤 2：重启后端服务
```bash
# 停止当前服务 (Ctrl+C)
# 重新启动
python -m uvicorn app.main:app --reload
```

### 步骤 3：验证加载
访问 http://localhost:8000/api/spiders 查看爬虫列表

**自动生成的配置**：
```json
{
  "my_spider": {
    "name": "my_spider",        // 默认使用文件名作为名称
    "enabled": true,
    "source": "unknown",
    "script_url": "",
    "file_path": "spiders/my_spider.py"
  }
}
```

## 方法二：添加文件并配置名称（推荐生产使用）

### 步骤 1：添加爬虫文件
```
spiders/
├── wawa.py          # 哇哇影视爬虫
└── ckzy.py          # CK 资源爬虫
```

### 步骤 2：编辑 spiders.json

**⚠️ 重要：JSON 对象的键就是爬虫的 key！**

```json
{
  "wawa": {                    ← 这个 "wawa" 就是 key（必须与文件名一致）
    "name": "哇哇影视",         ← 爬虫的显示名称
    "enabled": true,
    "source": "upload",
    "script_url": "",
    "file_path": "spiders/wawa.py"
  },
  "ckzy": {                    ← 这个 "ckzy" 就是 key（必须与文件名一致）
    "name": "CK资源",
    "enabled": true,
    "source": "upload",
    "script_url": "",
    "file_path": "spiders/ckzy.py"
  }
}
```

**关键规则**：
- ✅ **key 必须与文件名一致**：`wawa.py` → key 是 `"wawa"`
- ✅ **key 不能重复**：每个爬虫必须有唯一的 key
- ✅ **key 是爬虫的唯一标识**：用于 API 调用和爬虫管理

### 步骤 3：重启服务或调用重载 API
```bash
# 方法 1：重启服务
python -m uvicorn app.main:app --reload

# 方法 2：调用重载 API（无需重启）
curl -X POST http://localhost:8000/api/spiders/wawa/reload
curl -X POST http://localhost:8000/api/spiders/ckzy/reload
```

### 工作流程

#### 1. 克隆仓库
```bash
git clone https://github.com/your-username/LongTV-backend.git

```

#### 2. 添加新爬虫
```bash
# 创建爬虫文件
cd app/spiders
cat > new_spider.py << 'EOF'

```

#### 3. 更新配置文件
编辑 `spiders.json`：
```json
{

    "source": "upload",
```

```bash
git add app/spiders/new_spider.py
```

```bash
# 在服务器上
cd /path/to/LongTV-backend

# 重启服务或调用重载 API
systemctl restart longtv-backend
# 或
curl -X POST http://localhost:8000/api/spiders/new_spider/reload
```

## 配置字段说明

### spiders.json 字段

| 字段 | 类型 | 必需 | 说明 | 示例 |
|------|------|------|------|------|
| `name` | string | 是 | 爬虫显示名称 | "哇哇影视" |
| `enabled` | boolean | 是 | 是否启用 | true / false |
| `source` | string | 是 | 来源类型 | "upload" / "remote" / "unknown" |
| `script_url` | string | 否 | 远程脚本 URL | "https://..." |
| `file_path` | string | 是 | 文件路径 | "spiders/wawa.py" |

### source 字段说明

- **upload**: 本地上传的爬虫
- **remote**: 远程下载的爬虫
- **unknown**: 自动检测的爬虫（手动添加）

## 常见问题

### Q1: 添加文件后看不到爬虫？
**A**: 检查以下几点：
1. 文件名是否正确（不含中文和特殊字符）
2. 文件是否包含 `Spider` 类
3. Spider 类是否实现了必需方法
4. 是否重启服务或调用重载 API

### Q2: 如何批量导入多个爬虫？
**A**: 
```bash
# 复制所有 .py 文件到 spiders 目录
cp /path/to/spiders/*.py Fastapi/app/spiders/

# 重启服务
systemctl restart longtv-backend
```

### Q3: 如何禁用某个爬虫？
**A**: 编辑 `spiders.json`，设置 `enabled: false`：
```json
{
  "old_spider": {
    "name": "旧爬虫",
    "enabled": false,
    ...
  }
}
```

### Q4: 如何更新爬虫代码？
**A**: 
1. 直接修改 `.py` 文件
2. 调用重载 API：`POST /api/spiders/{key}/reload`
3. 无需重启整个服务

### Q5: 多个服务器如何同步爬虫？
**A**: 
1. 使用 Git 仓库管理
2. 各服务器定期 `git pull`
3. 调用 API 重新加载爬虫

## 自动化脚本示例

### 批量重载所有爬虫
```bash
#!/bin/bash
# reload_all_spiders.sh

BASE_URL="http://localhost:8000"

# 获取所有爬虫
SPIDERS=$(curl -s $BASE_URL/api/spiders | jq -r '.data[].key')

# 逐个重载
for key in $SPIDERS; do
    echo "重新加载: $key"
    curl -X POST $BASE_URL/api/spiders/$key/reload
    echo ""
done

echo "所有爬虫已重新加载"
```

### 监控爬虫文件变化自动重载
```bash
#!/bin/bash
# watch_spiders.sh

SPIDERS_DIR="Fastapi/app/spiders"

inotifywait -m -e modify,create,delete $SPIDERS_DIR |
while read path action file; do
    if [[ "$file" == *.py ]]; then
        key="${file%.py}"
        echo "检测到文件变化: $file"
        curl -X POST http://localhost:8000/api/spiders/$key/reload
    fi
done
```

## 最佳实践

1. **命名规范**：
   - 文件名使用小写字母和下划线：`movie_spider.py`
   - 避免中文和特殊字符
   - 保持 key 简洁有意义
   - ⚠️ **key = 文件名（不含 .py）**：`wawa.py` 的 key 是 `wawa`

2. **Key 的重要性**：
   - **Key 是爬虫的唯一标识**，用于所有 API 操作
   - **Key 必须与文件名一致**：如果文件是 `wawa.py`，key 必须是 `"wawa"`
   - **Key 不能重复**：每个爬虫必须有唯一的 key
   - **示例**：
     ```json
     {
       "wawa": {           ← key 是 "wawa"
         "name": "哇哇影视", ← 显示名称可以中文
         ...
       }
     }
     ```

3. **版本控制**：
   - 使用 Git 管理所有爬虫文件
   - 提交时附带清晰的说明
   - 定期备份 `spiders.json`

4. **测试流程**：
   - 本地测试通过后再部署
   - 使用 `/api/spider/{key}` 测试单个爬虫
   - 检查日志输出确认加载成功

5. **安全考虑**：
   - 不要在爬虫文件中硬编码敏感信息
   - 使用环境变量存储 API 密钥
   - 限制爬虫文件访问权限

6. **性能优化**：
   - 禁用不常用的爬虫
   - 定期清理无效爬虫文件
   - 监控爬虫执行性能

## 常见错误

### ❌ 错误 1：Key 与文件名不一致

```json
{
  "wawa_video": {          ← 错误：key 是 "wawa_video"
    "name": "哇哇影视",
    ...
  }
}
```

**问题**：文件名是 `wawa.py`，但 key 是 `wawa_video`，导致找不到爬虫。

**正确做法**：
```json
{
  "wawa": {                ← 正确：key 与文件名一致
    "name": "哇哇影视",
    ...
  }
}
```

### ❌ 错误 2：缺少 Key

```json
[
  {
    "name": "哇哇影视",     ← 错误：使用了数组，没有 key
    "enabled": true,
    ...
  }
]
```

**问题**：`spiders.json` 应该是对象 `{}`，不是数组 `[]`，必须有 key。

**正确做法**：
```json
{
  "wawa": {                ← 正确：添加了 key
    "name": "哇哇影视",
    ...
  }
}
```

### ❌ 错误 3：Key 重复

```json
{
  "wawa": {
    "name": "哇哇影视",
    ...
  },
  "wawa": {                ← 错误：key 重复
    "name": "另一个爬虫",
    ...
  }
}
```

**问题**：JSON 对象的键必须唯一，重复的 key 会被覆盖。

**正确做法**：使用不同的 key：
```json
{
  "wawa": {
    "name": "哇哇影视",
    ...
  },
  "wawa2": {               ← 正确：使用不同的 key
    "name": "另一个爬虫",
    ...
  }
}
```

### ❌ 错误 4：Key 包含中文或特殊字符

```json
{
  "哇哇影视": {            ← 错误：key 包含中文
    "name": "哇哇影视",
    ...
  }
}
```

**问题**：Key 应该使用小写字母、数字、下划线，避免中文和特殊字符。

**正确做法**：
```json
{
  "wawa": {                ← 正确：使用英文 key
    "name": "哇哇影视",     ← 显示名称可以中文
    ...
  }
}
```

## 故障排查

### 爬虫没有出现在列表中？

1. 检查 `spiders.json` 是否有效 JSON
2. 确认 key 与文件名一致
3. 检查 Python 文件是否有语法错误
4. 查看后端日志确认加载状态

### 保存名称失败？

**最常见原因**：`spiders.json` 中缺少 key 或 key 格式错误。

**解决步骤**：
1. 打开 `spiders.json`
2. 确保使用 `{}` 对象格式，不是 `[]` 数组
3. 确保每个爬虫都有唯一的 key
4. 确保 key 与 `.py` 文件名一致
